# Superpowers

`Superpowers` 用来保存工程决策、设计草案、历史分析、路线图和旧资料归档。这里的内容不会删除，也不会混入新用户的主阅读路径。

公开文档回答“用户如何理解、部署和使用 IoT DC3”。`Superpowers` 回答“项目为什么这样演进、哪些方案曾被讨论、哪些资料需要继续维护”。

## 分类

| 分类 | 目录 | 用途 | 阅读对象 |
|------|------|------|----------|
| 文档机制 | [documentation](documentation/) | 文档分层、维护规则、评审清单和发布验证 | 维护者、AI 助手 |
| 设计方案 | [design](design/) | 架构调整、领域模型、接口边界和分阶段落地方案 | 核心开发者 |
| 分析材料 | [analysis](analysis/) | 竞品/协议/技术路线分析，保留结论来源和时间 | 产品与技术决策者 |
| 策略草案 | [strategy](strategy/) | 项目叙事、AI 定位、对外表达草案 | 维护者 |
| 待办池 | [backlog](backlog/) | 未形成完整 issue 或设计文档的工程线索 | 维护者 |
| 旧资料归档 | [legacy-operation](legacy-operation/) | 旧版 UI 截图式操作说明，保留历史参考价值 | 迁移维护者 |

## 使用原则

- 公开文档优先保证准确、可执行、低认知负担。
- 工程推演、未决方案、历史截图和长期计划放在 `Superpowers`。
- 资料迁移到 `Superpowers` 时保留原始信息，但需要补充当前状态、时间背景或适用范围。
- 对外首页、快速开始、操作手册、架构和开发文档不得依赖 `Superpowers` 才能读懂。
- 涉及运行方式、环境变量、模块清单、驱动数量的内容必须以仓库当前文件为准。
