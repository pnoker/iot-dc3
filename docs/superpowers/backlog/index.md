# 待办池

这里保存尚未形成完整 issue、设计方案或开发任务的工程线索。条目进入开发前，应补充背景、影响范围、验收标准和对应模块。

## 配置常量治理

| 条目 | 背景 | 建议动作 | 状态 |
|------|------|----------|------|
| 分隔符常量统一 | 代码中可能存在 `"."`、`"#"` 等字符串分隔符直接散落使用的情况 | 全仓搜索常见分隔符和 routing key/topic 拼接逻辑，判断是否应集中到常量类或协议专用 helper 中 | 待梳理 |

## 处理规则

- 能在 1 个 PR 内修复的条目，拆成明确任务并移出待办池。
- 涉及协议、告警、状态、租户或消息路由的条目，需要先补设计方案。
- 已经落地的条目应补充对应 PR、commit 或文档链接。
